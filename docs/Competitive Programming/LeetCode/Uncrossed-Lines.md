# Uncrossed Lines - 不相交的线

## 分析
有趣的构造题（或许在有些人眼里不算构造题，但于我而言，只要不是上来就让你求出 XX 数组的 XX 值这样直白的题目，都属于构造题）。最近的定力很差，总是忍不住去看别的东西，所以这道题想了一个小时。

看到这道题的时候第一反应是求逆序对，因为之前在 cf 的某一场 Div. 4 里遇到过相似的题目，也是求线段相交什么的，但后来发现两者几乎没有关系，而唯一的关系在于 “相交” 的判定条件，结果证明，这是本题的突破口。

解编程题的本质是将问题转换为数学模型，然后利用算法求出该模型的可行解。为了对这个问题进行建模，我首先开始思考 “相交” 的意义，也就是前面所说的 “判定条件”。如果给你两条线段（在本题的情景下），如何判断它们是否相交？

联想到以前做的那道题，我很快就得到了答案：线段端点的先后顺序在两个数组中相反。

假设这两条线段的起点分别为 $i1$、$i2$，位于数组 A，终点分别为 $j1$、$j2$，位于数组 B，不失一般性，规定 $i1 < i2$，那么只有当 $j1>j2$ 时，两线段才是相交的。

反过来说，为了构造不相交的线段，我必须保证新增线段的两个端点和已有的任何端点之间均不符合此条件。

注意到条件中存在两个不等式，不太好利用，自然地，我想到从左往右枚举 A 数组，逐个选取元素作为线段的一端，这样就能保证 $i1<i2$ 这个不等式始终成立，转而去考虑剩下的不等式 $j1<j2$ （为了让线段不相交）。

为了让答案尽可能地大（不相交的线段尽可能多），这里存在贪心策略，即每次在 A 数组中选定线段的一端时，对于线段的另一端（位于 B 数组），如果存在多个候选元素，那么尽可能选下标最小的，这样才能保证后面的新增线段有更多的选择。

上述过程可以用 DFS 来描述，并通过引入 DP 数组实现记忆化搜索，将指数级别的时间复杂度优化至多项式级别。

## 代码
```cpp
int maxUncrossedLines(vector<int>& nums1, vector<int>& nums2) {
        std::map<int, std::vector<int>> m;
        int n1 = nums1.size(), n2 = nums2.size();
        for (int i = 0; i < n2; i++) {
            m[nums2[i]].push_back(i);
        }

        std::vector<std::vector<int>> dp(n1, std::vector<int>(n2, -1));

        auto dfs = [&](auto&& self, int k, int prev) {
            if (k == n1) {
                return 0;
            }

            if (prev != -1 && dp[k][prev] != -1) {
                return dp[k][prev];
            }

            int res = 0;
            res = std::max(res, self(self, k + 1, prev));
            for (auto idx : m[nums1[k]]) {
                if (idx > prev) {
                    res = std::max(res, 1 + self(self, k + 1, idx));
                    break;
                }
            }

            if (prev != -1) {
                dp[k][prev] = res;
            }

            return res;
        };

        return dfs(dfs, 0, -1);
    }
```
